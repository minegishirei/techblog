title:oo4oの移行について ##oo4o (Oracle Objects for OLE) は新しいバージョンで使えない 2oo4oの移行作業が各企業で始まっている。 原因はこれ 
 
Oracle Database 11gリリース2(11.2) は、Oracle Objects for OLEをサポートする最後の データベースバージョンです。
Oracle Database 11gリリース2(11.2)の後にリリースされるOracle Databaseバージョン では、 
Oracle Objects for OLEはサポートされません。
オラクル社のライフタイムサポート・ポリシーの対象となっている既存のOracle Databaseバージョンでは、 
Oracle Objects for OLEを引き続き使用できます。
要約すると ###oo4o (Oracle Objects for OLE)は新しいバージョンで使えないよ! と言っている。

oo4oとは? 「Oracle Objects for OLE」の略

 
Oracle Objects for OLE(oo4o)を使用すると、 
Microsoft COMオートメーションおよびActiveXテクノロジをサポートするプログラミング 
またはスクリプト言語を介して、Oracleデータベースに格納されたデータにアクセスでき ます。 
これには、Visual Basic、Visual C++、Visual Basic for Applications (VBA)、
IIS Active Server Pages(VBScriptおよびJavaScript)などが含まれます。
ある特定の言語(Microsoft COMおよびActiveXをサポートする言語)は COMまたはDCOMと呼ばれるMicrosoftが用意した技術を使って oo4oというOracleが用意してくれているサーバーにアクセスできる。 そのoo4oはOracleデータベースに接続することができる。 まとめると ###oo4oを使うとオラクルデータベースに接続できた ということ。 ※以上は全部Microsoftのお話し。 title:oo4oの移行について description:oo4o (Oracle Objects for OLE) は新しいバージョンで使えない 2oo4oの移行作業が各企業で始まっている。 img:https://ka-chanblog.com/wp-content/uploads/2020/10/Oracle-Logo.png
